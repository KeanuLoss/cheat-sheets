Compiled / Interpreted languages

Some programming languages use a compiler.

The code is sent to the compiler, which translates the entire program into code that is compatible with the target or host machine

the target machine executes the code and returns the output


Interpreted languages load an interpreter and return the output without creating executable machine code as an independent artifact

The interpreter reads the code (checks if it is reading a valid Python command), translates the code into byte code and then executes it in a Python virtual machine which returns the output

Floating point numbers



To use decimal number you need to use floating-point numbers ( float )

Strings

Text is stored in string data type ( str )

strings represent a series of characters, including letters, numbers, punctuation marks and spaces

Strings are always in quotes " " or single quotes ' '

strings are immutable, which means you cannot change the string that a variable contains

certain characters can't be typed into a string:
If you need quotation marks inside your string they would end the string -> you need to use an " escape character, a back slash " \ "

the combination of the escape character with the quotation mark is called an escape sequence

if you need a backslash in a string you need to use two back slashes

if you want to create a new line you need to use a new line character " \ n "
this will place the cursor / following text in a new line

if you want to use a tab-character you need to use " \ t "

if you want to return the cursor to the beginning of the line and overwrite the string you need to use a carriage return character use " \ r "

You can ignore the escape character by using a raw string that starts with an " r " before the quotation mark

the string that follows will ignore any escape sequences

if you want to denote text that you want to span multiple lines without using new line characters you can use triple quotes

You create a set by setting a variable equal to curly braces

the items in a set can be strings or any kind of numbers

Set methods

Files

It's important to save data to the hard disk or other permanent storage so you can retrieve that document later.

File options

" x " = create -> will create a new file with the specified file name, if the file already exists, you will get an error message

" r " = read -> opens a file for reading

" a " = append -> will create a new file if it doesn't exist yet
if the file already exists, it will open the file and append the contents to the end of the file

" w " = write -> will create a new file if it doesn't exist yet
if the file already exists, it will erase all contents of the existing file and start writing at the beginning of the file

Writing to a file

In order to write data to a file you first have to open that file

You open a file by using the " open " - method, specifying a filename and an option, depending on what you want to do with that file

Once the file is open you can start writing using the " .write "-method.
example : " file-name.write( " Content you're writing " ) "

when you're done writing always make sure to close the file with " file-name.close( ) "

When you can't write to a file it's because it is " file locked "

Developers use a " file lock " to prevent multiple applications to write to the same file at the same time

Reading from a file

In order to read from a file you first have to open it and use the read-option.

" print(variable.read ( ) ) "

Assignment operators

  • " = " assigns the value on the right to the variable on the left
  • " += " takes the value currently stored in a variable, adds the value on the right to it and assigns the value back to the variable
  • " -= " takes the value currently stored in a variable, subtracts the value on the right from it and assigns the value back to the variable
  • " *= " takes the value currently stored in a variable, multiplies it by the value and assigns the new value back to the variable
  • " /= " takes the value currently stored in a variable, divides it by the value and assigns the value back to the variable
  • " %= " takes the value currently stored in a variable, computes the modulus of it and the value on the right and assigns the new value back to the variable
  • " **= " takes the value currently stored in a variable, raises it to the power of the value on the right and assigns the new value back to the variable
The operations follow mathematical orders when calculating the value of an expression.

Loops

Range function

the range function creates a range of numbers based on a set of parameters :

1. start (optional) -> tells the range function at which number to start
its optional, if you omit this parameter the starting point is 0

2. stop (required) -> tells the range function at which number you want to stop NOTICE : this number will not be included

3. step (optional) -> tells the range function how many numbers to skip between each number in the set

the parameters are seperated by commas
example : " for variable in range(2,10,1) "
this can be used to iterate through lists by iterating through the indexes

Scope

Scope is the sections within code for which a variable or function is defined and executable for the interpreter

Scope refers to the definition of variables or functions in Python.

A variable / function is out of scope when it is used / called before it has been assigned / defined

variables defined within functions only have scope within the function they were assigned in

when you define a function within another function ( nested function ), the nested function only has a scope of its partent function
so it can only be called from inside the parent function, not globally

global variables

by using " global variable-name " within a function, you can use the global variable instead of creating a new local version

by using " nonlocal " in a nested function you tell the interpreter that you don't want to create a local variable and instead want to use that variable in the parent function

Arguments

the order you pass arguments to the function is the order those arguments are assigned to the parameters

when you call the arguments specifically by name inside the function call you can specify them in any order you want.
example : function-name( paramer1 = argument, parameter2 = argument, ... )

Errors

Syntax Errors

Exception Handling

sometimes code that conforms to Python programming rules and that does not cause a syntax error still breaks during execution

exceptions occur when code is properly formed syntactically but an error occurs that causes the program to terminate

an exception will cause the application to crash and cease execution

ZeroDivisionErro = program tried to divide by zero

FileNotFoundError = file that was tried to be opened, read or written can't be found

If you ever come across an error that you don't understand, you can simply search for that error on the internet

Avoiding exceptions

The trick is to anticipate where exceptions can possibly happen and code in a way that will not make your application crash

1. use the " try " - statement preceding any block of code in which we think an exception may occur
example : " try : indented block of code"

any code in the indented block that follows the try statement will be set up for the proper handling of exceptions

2. use the " except " clause after the try block of code followed by a type of error
example : " except ZeroDivisionError / any other possible error "
you don't need to specify the error type and can just use " except : "

if the specified error occurs, the program will jump to the except block instead of crashing

the indented code after the except clause will be executed, which should be an explanation what went wrong and what input should be put in

either way (if the try block was succesfully executed or the exception block was executed) the program will resume normal operation

it's good to use a " finally "-clause in order to run a block of code regardless of errors to release resources / clean up temporary data / release locks / ensure user notification

even when an exception is raised the finally block will still be executed

you can raise errors with " raise " + any error

Logs

Logical errors are most difficult to find because the interpreter doesn't point to the likely location of the error

You need to compare the obtained results to the expected results to make sure the program is doing what you intended it to do

the process of identifying and fixing logical errors is called debugging

logging / writing data to log files is a tool for debugging

you have to import the library " logging "

this lets you write code using Pythons built in logging features

logging in Python has five different levels of severity in terms of which information is being logged

in order from lowest to highest severity :
1. debug
2. info
3. warning
4. error
5. critical

you can log messages through one of those severity levels

you can set a severity level for your program so that during execution, only the severity levels equal to or higher than will be logged

with " logging.basicConfig ( ) " you can change the level of log outputs as well as write logs to files
with " logging.basicConfig ( level = logging.SEVERITY-LEVEL ) " you can change the severity level

with " logging.basicConfig (filename = "mylog.log", filemode = "w", level = logging.SEVERITY-LEVEL ) you can write the logs to a file

you can customize your log messages in order to get more detailed info on what is going on in your program by using the format-string
by using " logging.basicConfig ( format = "%( attribute )s: %(message)s " ) "

here is a list of attributes to pull different data in your logs:
1. " %(asctime)s " - gets the log entry creation date and time
2. " %(filename)s " - gets the name of the file
3. " %(funcName)s " - gets the name of the function in which the log entry was created
4. " %(lineno)d " - gets the line of code where the log entry was created
5. " %(process)d " - gets the process ID that created the log entry
6. " %(processName)s " - gets the process name that created the log entry
