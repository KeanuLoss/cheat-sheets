Decorators

Decorators are functions which modify the functionality of other functions and help to make the code shorter.

They are useful if you want to use functions on multiple other functions

With a decorator you only need to define the wrapper function once and then can decorate every other function with it

These are the steps you need to take :

1. create a decorator-function-name(function-name of the function that should be wrapped)

2. define wrapper function within decorator function : def wrapper(parameter)

3. place function you want to be wrapped inside the wrapper function

4. return wrapper-function

5. " @decorator-function-name( ) def wrapped-function-name( ): function body "

6. call the decorated function " wrapped-function-name( )"

Multiprocessing

You need to import the " multiprocessing " module in order to create processes and to use shared memory for IPC

Create a process-object by assigning " process-name = multiprocessing.Process( target = function-name , args = ( parameters, ) ) "

You start the process-object with " process-name.start() "

Every process has its own adress space (virtual memory), this means you can't store the process results in a global result

If you want to share data between processes you need to use interprocess communication (IPC) techniques

Multiprocessing locks

Locks are required to ensure that a process is fully executed before another can access that share value and change it.

1. create a lock variable " lock-variable = multiprocessing.Lock( ) "

2. pass that lock-variable to the process-arguments and the function parameters

3. inside the functions : use " lock.acquire( ) " before the change to the shared value

4. inside the functions : use " lock.release( ) " after the change to the shared value

Multiprocessing pool

Distributing a task to multiple cores of your CPU is called parallel processing

Dividing the task to the different cores is called " map "

Aggregating the results together is called " reduce "

2. create a pool object " p = Pool( )"

3. use " result = p.map( function-name,variable or range you want to divide ) "

5. use " p.join( ) "

NOTICE :pool is only faster when a lot of computational tasks are required

When you use " yield " it produces values only one at a time

CONS : generators can only iterate once - if you need to re-iterate you need to recreate the generator and " reset " the yield-value.

Setting up the database

1. create a database.py 2. create a models.py 3. create a crud.py 4. create a schemas.py

First you need to set up the schemas in your schemas file, which will validate that the input data is in the right format in order to be processed.

You have to create BaseModel classes for each schema in order to add data to a database.

The next step is to create the crud methods that will add the task (the text that needs to be translated & the language it's going to be translated in) and updates the task once it has been translated.

Next you need to set up the database and create the table.

For this you need to import os, from sqlalchemy import create_engine, from sqlalchemy.orm import sessionmaker, from dotenv import load_dotenv

inside your .env file you have your sensitive data like DATABASE_URL & API_KEY.

You need to load_dotenv().

create a get_db function: def get_db():
db = SessionLocal()
try:
yield db
finally:
db.close()

SQLALCHEMY_DATABASE_URL = os.getenv("DATABASE_URL")

engine = create_engine(SQLALCHEMY_DATABASE_URL)

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

Next you need to define the table in the models file.

from sqlalchemy import Column, Integer, String, Text, JSON

from sqlalchemy.ext.declarative import declarative_base

Base = declarative_base()

Create a Base class with all the columns that are necessary.
Set the attributes (id/status/...) = Column-function(parameters)

Enter a name for the server

Enter the Host name/adress of your database & your postgres password

"Databases" -> "Schemas" -> Tables
In " Tables " you create a table with the class in your models file

in order to connect your database to the specific project you need to set up the DATABASE_URL inside your .env file :

" DATABASE_URL=postgresql://username:password@localhost:5432/database name "
postgresql = what SQL (in case you don't use postgresql)
